029 — RAG OBSERVABILITY & TRACING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

WHY OBSERVABILITY?
  Without tracing, a wrong answer could be a retrieval failure, a generation
  failure, or a query failure — and you can't tell which.
  Observability gives you per-hop visibility to find and fix the right layer.

─────────────────────────────────────────
KEYWORDS
─────────────────────────────────────────

trace              → full record of one query's journey through the pipeline
span               → one unit of work within a trace (e.g. retrieval, LLM call)
callback handler   → LangChain hook: on_llm_start, on_llm_end, on_chain_start
LangSmith          → hosted tracing by LangChain team (set env vars, auto-traced)
Arize Phoenix      → open-source, self-hosted, OpenTelemetry-based RAG tracer
OpenTelemetry      → standard for distributed tracing (spans, traces, metrics)
retrieval score    → similarity score for each retrieved chunk (lower = better for FAISS L2)

─────────────────────────────────────────
5 HOPS TO TRACE
─────────────────────────────────────────

1. QUERY     → original question + query embedding
2. RETRIEVAL → chunks returned + their scores + latency
3. CONTEXT   → assembled context size (chars / tokens)
4. GENERATION→ prompt sent + response + token count + latency
5. ANSWER    → final output + faithfulness check

─────────────────────────────────────────
LANGCHAIN CALLBACK PATTERN
─────────────────────────────────────────

from langchain.callbacks.base import BaseCallbackHandler

class MyTracer(BaseCallbackHandler):
    def on_llm_start(self, serialized, prompts, **kwargs):
        self.t0 = time.time()
    def on_llm_end(self, response, **kwargs):
        self.latency = time.time() - self.t0

tracer = MyTracer()
llm = ChatGroq(..., callbacks=[tracer])

─────────────────────────────────────────
PRODUCTION TOOLS
─────────────────────────────────────────

LangSmith  → LANGCHAIN_TRACING_V2=true + LANGCHAIN_API_KEY
             Automatic for all LangChain code. Hosted UI.

Arize Phoenix → pip install arize-phoenix
                px.launch_app()  # localhost:6006
                register()       # auto-instruments OpenAI/LangChain/LlamaIndex

Custom JSON log → BaseCallbackHandler → log to file/DB
                  Full control, no external dependency

─────────────────────────────────────────
KEY METRICS
─────────────────────────────────────────

retrieval_latency_s     → how fast is the vector DB?
generation_latency_s    → how fast is the LLM?
top_retrieval_score     → are we finding relevant chunks?
context_chars           → are we sending too much / too little to the LLM?
answer_faithfulness     → is the answer grounded in the context? (use RAGAS)
